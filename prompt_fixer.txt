# Bug Research Report: Issue #284
## [CI-BUG] README in templates/enterprise-gke missing Architecture header

### 1. Root Cause Analysis
The issue was caused by a combination of a systemic flaw in the project's linter and an accidental file corruption by an automated agent.

- **Systemic Linter Flaw:** The script `agent-infra/local-lint.sh` was using piped `while read` loops (e.g., `find ... | while read ...`). In Bash, this construct creates a subshell for the `while` loop. Any `exit 1` called within this loop would only terminate the subshell, allowing the main script to continue and ultimately exit with code 0 (success). This meant that linting failures (such as missing mandatory headers or misplaced CI markers) were not blocking merges.
- **Agent Corruption:** An automated agent working on a previous issue (likely #255 or an early attempt at #284) accidentally overwrote the entire content of `templates/enterprise-gke/README.md` with its own completion report ("I have completed the fix..."). 
- **Consequence:** Because the linter failed to block the corrupted file, it was merged into the repository. The missing `## Architecture` header was a direct result of the entire file being replaced by non-documentation text.

### 2. Affected Files
- `templates/enterprise-gke/README.md`: Corrupted with agent status text, missing all standard sections.
- `agent-infra/local-lint.sh`: Contained the subshell bug that prevented linter enforcement.
- `templates/basic-gke-hello-world/README.md`: Contained a manual "Validation Record" section after the CI marker, which violated the mandate that the CI marker must be on the last line.

### 3. Similar Errors and Solutions
- **Issue #255:** Also reported corruption in the same README, but the fix attempt was itself corrupted or blocked by the linter bug.
- **Solution:** The standard solution for the "exit in subshell" problem is to avoid pipes to `while read` and instead use `for` loops or process substitution.
